1. Remove mongo-migrate as a dependency.  It is a legacy from a previous unsuccessful attempt to create a true data migration script.

2. Create a migration that works like Flyway or Liquibase for Java (this application is Python/MongoDB/Beanie).  It should assure that the migration runs at startup, first checks to see if the migration has been run previously, and only runs the migration if it has never been run previously.  It can and should leave a record of the migration in MongoDB so that it can check for prior successful completions.  This can be done using custom code or by using a well-regarded and stable migration project.

3. The migration should run when this service is first started if the migration has not previously been run successfully.  The migration should run before the service becomes available.
